# Frontend UX → moved

The Frontend UX standards now live in **`sneat-specs`**, the cross-repo home for
Sneat standards, because they span every extension repo (`calendarius`,
`contactus`, `listus`, …), not just `sneat-libs`.

➡️ **[sneat-specs / standards / frontend-ux](https://github.com/sneat-co/sneat-specs/blob/main/standards/frontend-ux/README.md)**

What you'll find there:

- **cards** — `ion-card` as the content container, header styles.
- **card-title-buttons** — action buttons in card / section headers.
- **buttons** — colour, fill, expand, icon, and in-flight conventions.
- **lists** — `ion-list`, sliding items, grouped/collapsible lists.
- **page-layout** — header/toolbar, content, footers, segments.
- **forms** — inputs, labels, reactive forms, validation, submit.
- **modals** — modals, popovers, confirms, toasts.
- **states** — loading, empty, and error states.

This remains the **UX** pillar of the
[Sneat extension standards](../README.md) (backend · frontend apps · UX).
